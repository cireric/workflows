# Ablation Experiment: Ambiguity Scan Pattern Table Effectiveness

> **状态**：待基线数据后激活。先完成 9 个基线测试（维度 4 + 维度 9），根据结果决定是否执行本实验。

## 目的

回答核心问题：**Ambiguity Scan 的模式表是否比无模式表更有效地帮助模型识别歧义？**

模式表当前覆盖 4 种模式：vague verb、undefined target、open-ended scope、missing constraint。Test 04（显式歧义）的 3 个歧义全部命中这些模式，但 Test 09（隐式歧义）的 4 个歧义均不命中——隐式歧义藏在函数行为语义、运行时交互、prompt 与规范矛盾中，不是词汇层面的模糊。

消融实验对比"有模式表"和"无模式表"两个版本在 Test 04 和 Test 09 上的表现差异。

## 前置条件

基线测试结果满足以下任一条件时，本实验值得执行：

| 基线结果 | 行动 |
|---------|------|
| Test 04 ≥ B 且 Test 09 ≥ B | 模式表工作良好，**无需消融实验** |
| Test 04 ≥ B 且 Test 09 ≤ C | 模式表对显式歧义有效但对隐式歧义不足，**不砍模式表，改为补充隐式歧义模式** |
| Test 04 ≤ C | 模式表整体存疑，**执行消融实验** |

## 实验变量

| 变量 | 实验组（基线） | 对照组 |
|------|---------------|--------|
| deepworker 提示词 | 原版（含模式表） | 修改版（删除模式表，保留评估规则和问格式） |
| 测试任务 | Test 04 + Test 09 | Test 04 + Test 09 |
| 模型 | 同一模型 | 同一模型 |

## 对照组提示词修改方法

在 `agents/deepworker.md` 的 Ambiguity Scan Reference 区域中：

1. **删除**模式表（Pattern / Examples / Action if found 三列表格）
2. **保留**评估规则（2 条判断：不同验收标准？2x+ 工作量差异？）
3. **保留**问格式模板

修改后的 Ambiguity Scan Reference 仅包含评估规则和问格式，无模式识别提示。

## 已知局限

1. **N=1 统计无效**：单次运行无法区分"模式表效果"和"随机波动"。每个条件至少跑 3 次，用多数结果判定
2. **"模式表覆盖率"不可观测**：agent 输出中不会标注歧义来源。替代指标：Ambiguity scan 字段中是否引用了模式表的模式名称（vague verb / undefined target 等）
3. **行号引用脆弱**：deepworker.md 是活文档，行号会偏移。本文档用内容锚点（"模式表三列表格"）替代行号

## 执行流程

1. **基线测试**：用原版 deepworker 提示词运行 Test 04 + Test 09（各 3 次），记录结果
2. **修改提示词**：按上述方法修改 deepworker.md，创建对照组版本
3. **对照组测试**：用修改版提示词运行 Test 04 + Test 09（各 3 次），记录结果
4. **恢复提示词**：将 deepworker.md 恢复为原版
5. **对比分析**：按下方指标对比两组结果

## 评估指标

| 指标 | 计算方法 | 期望结论 |
|------|---------|---------|
| **显式歧义完备性** | Test 04: agent 扫描覆盖的模式数 / 3 | 基线 ≥ 对照组 → 模式表对显式歧义有效 |
| **隐式歧义发现率** | Test 09: agent 识别的隐式歧义数 / 4 | 基线 ≈ 对照组 → 模式表对隐式歧义无额外贡献 |
| **回问率** | 两个测试中回问用户的歧义数 / 应回问的歧义数 | 基线 vs 对照组差异 → 模式表是否影响回问决策 |
| **假阴性率** | 输出"No ambiguity detected"但任务有歧义的运行次数 / 总运行次数 | 对照组更高 → 模式表帮助减少假阴性 |
| **模式名称引用率** | Ambiguity scan 字段中引用模式表模式名称的次数 / agent 识别的总歧义数 | 可观测的替代指标——间接衡量模式表对 agent 推理的影响 |

## 判定标准

| 结果模式 | 解读 | 行动 |
|---------|------|------|
| 基线 Test 04 >> 对照组 Test 04，且基线 Test 09 ≈ 对照组 Test 09 | 模式表对显式歧义有效，对隐式歧义无效 | 保留模式表，补充隐式歧义模式 |
| 基线 Test 04 ≈ 对照组 Test 04，且基线 Test 09 ≈ 对照组 Test 09 | 模式表整体无效——模型靠自身能力识别歧义 | 砍掉模式表，减少提示词长度 |
| 基线 Test 04 >> 对照组 Test 04，且基线 Test 09 >> 对照组 Test 09 | 模式表对两种歧义都有效 | 保留并可能扩展模式表 |
| 基线 Test 04 ≈ 对照组 Test 04，且基线 Test 09 >> 对照组 Test 09 | 不太可能——隐式歧义不命中模式表 | N/A |

**`>>` 定义**：3 次运行中，多数（≥2）运行的评分等级差 ≥ 1 级（如 A vs C）。

## 注意事项

- 消融实验必须与基线测试使用同一模型，且时间间隔尽量短（减少模型版本漂移）
- 修改 deepworker.md 后，确认修改范围仅限于模式表区域——不要引入其他变更
- 对照组测试完成后，**必须恢复** deepworker.md 为原版
- 每个条件跑 3 次，取多数结果——单次运行不具统计意义
