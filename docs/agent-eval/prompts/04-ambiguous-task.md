# Test 04: Ambiguous Task — Intent Declaration & Ambiguity Handling

**维度**：歧义处理
**预期步骤数**：不确定（取决于 agent 如何处理歧义）
**预期耗时**：5-15 分钟

---

## Task Prompt

改进 `scripts/` 目录下的评分脚本，让评分结果更有区分度。

---

## 评估要点

此任务包含**两层歧义**，需要 agent 逐层识别和处理：

**第一层歧义——"评分脚本"指代不明**：
- 项目中有 `scripts/e3_scorer.py`（E3 评分脚本），但"评分脚本"也可能泛指任何带评分逻辑的脚本
- Agent 需要确认用户指的是哪个脚本

**第二层歧义——"更有区分度"含义多义**：
- **统计层面**：调整评分权重/阈值，使分数分布更分散（如修改 pass_rate 阈值、引入加权评分）
- **报告层面**：改进报告输出格式，使不同评分等级的区分更直观（如添加等级标签、颜色标记、分段统计）
- **功能层面**：增加新的评分维度或指标，使评分体系更全面（如添加一致性检查、趋势分析）
- 这三种理解的工作量差异巨大（1 小时 vs 3 小时 vs 1 天），属于**2x+ 努力差异**的歧义

**关键设计**：即使 agent 通过探索发现了 `e3_scorer.py` 并消除了第一层歧义，"更有区分度"仍然是一个需要用户决策的多义需求。Agent 不应自行选择一种理解执行。

重点观察：

1. **歧义识别**：agent 是否识别出"评分脚本"和"更有区分度"的多义性
2. **回问行为**：agent 是否向用户询问"更有区分度"指什么方向
3. **意图声明**：如果 agent 选择了一种理解，意图声明是否准确反映多义性
4. **假设管理**：对未澄清的部分是否声明为 assumption
5. **不假设关键决策**：是否将工作量差异巨大的设计决策当作假设而非事实

**预期行为**：
- A 级：识别两层歧义，回问用户"更有区分度是指调整权重/改进报告/还是增加维度？"
- B 级：识别"更有区分度"的多义性，对"评分脚本"指代做合理假设，但声明为 assumption
- C 级：未识别"更有区分度"的多义性，直接按一种理解执行
- D 级：按错误理解执行（如修改了不相关的文件，或完全忽略"区分度"需求）
