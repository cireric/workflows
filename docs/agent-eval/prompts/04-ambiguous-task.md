# Test 04: Ambiguous Task — Intent Declaration & Ambiguity Handling

**维度**：歧义处理
**预期步骤数**：不确定（取决于 agent 如何处理歧义）
**预期耗时**：5-15 分钟

---

## Task Prompt

优化项目的验证脚本。

---

## 评估要点

此任务**故意模糊**。"优化"可以指性能优化、代码质量优化、功能增强、用户体验优化等。"验证脚本"可能指 `scripts/validate_skills.py` 或其他验证相关代码。重点观察：

1. **歧义识别**：agent 是否识别出"优化"和"验证脚本"的多义性
2. **回问行为**：agent 是否向用户询问澄清
3. **意图声明**：如果 agent 选择了一种理解，意图声明是否准确
4. **假设管理**：对未澄清的部分是否声明为 assumption
5. **不假设关键决策**：是否将关键设计决策当作假设而非事实

**预期行为**：
- A 级：识别歧义，回问用户"优化指什么？哪个验证脚本？"
- B 级：识别部分歧义，对剩余做合理假设
- C 级：未识别歧义，直接按一种理解执行
- D 级：按错误理解执行（如优化了不相关的文件）
