# ADR 0007: Deepworker 混合提示词架构（原则驱动 + 指令跟随 + 约束重注入）

## Status

Accepted

## Context

设计 Deepworker（深度执行 Agent），参考 OmO 的 Hephaestus。Hephaestus 针对GPT-5.x采用纯原则驱动提示词——给目标不给方法，信任模型自主规划执行路径，24小时自主运行不衰减。

Deepworker 的目标模型是 K2.6（备选 GLM-5.1 / DeepSeek-V4-Pro）。三个模型有共同弱点：**长会话中约束衰减**。

| 模型 | 衰减表现 |
|---|---|
| K2.6 | v1.41 后质量回退，discard-all 上下文策略丢失早期状态 |
| GLM-5.1 | SSE 截断 bug，~100K 后输出质量下降 |
| V4-Pro | WDCD -100% 指令衰减（Round 3 后约束完全丢失） |

## Decision

采用**混合架构**：核心原则层（原则驱动）+ 执行协议层（指令跟随）+ 约束重注入机制（补偿衰减）。

具体设计：
1. **身份与目标**用原则驱动——利用 K2.6 的自主规划能力
2. **操作循环、失败恢复、完成标准**用明确协议——防止模型跳过关键步骤
3. **约束重注入**在阶段转换时显式声明——补偿 WDCD 衰减
4. **TODO 铁律**作为执行基础纪律——100% 覆盖，始终生效
5. **TDD**作为可选增强层——适用时叠加，不适用时不叠加

## Alternatives Considered

### 纯原则驱动（Hephaestus 方案）

简洁提示词，依赖模型自主决策。

**优点**：提示词短，token 效率高，充分利用模型推理能力。
**缺点**：K2.6 在 Round 3 后约束衰减 -100%——纯原则驱动意味着模型既忘了约束，也忘了"要遵守约束"这条规则本身。自指规则在指令衰减时最先失效。
**结论**：不安全。GPT-5.5 能维持约束 24 小时，K2.6 做不到。

### 纯指令跟随

每步写死，检查清单式提示词。

**优点**：最安全，最小化模型自主决策空间。
**缺点**：浪费 K2.6 的自主规划能力（SWE-Bench Pro 58.6%），把能自主思考的模型降级为脚本执行器。
**结论**：过度保守。模型有自主规划能力，应该利用。

### 混合架构（选定方案）

在两者之间取平衡——核心利用模型能力，关键环节用协议保护。

**优点**：兼顾模型能力和可靠性。原则驱动层利用 K2.6 自主规划，协议层补偿衰减。
**缺点**：提示词比纯原则驱动长，token 开销增加。需要在"足够结构"和"不过度仪式化"之间持续校准。

## Consequences

- 提示词长度约为 Hephaestus GPT-5.5 提示词的 60-70%（混合架构的协议层开销低于 GPT-5.5 的多版本提示词，但高于纯原则驱动）
- 约束重注入在每次阶段转换时额外消耗约 50-100 tokens（硬约束摘要 + 任务约束）
- TODO 铁律的维护开销（标记 in_progress/completed）约占总输出的 10-15%
- 如果未来 K2.7 或更后版本解决了指令衰减问题，可以逐步减少协议层，向纯原则驱动迁移
- ADR 0004（冗余消除原则）仍然适用：如果约束重注入被证明在特定场景下冗余，应精简而非保留
