# ADR 0009: PLAN 执行不可逆性——信任光谱修正

## Status

Accepted

## Context

ADR 0008 将 PLAN 阶段定位为"可逆，偏 Hephaestus"（现称偏自主判断），理由是"模型规划能力是强项，漂移检测已提供安全网"。

5 组对照实验（T1-T5）提供了新证据：

指令跟随率较低的模型在 5/5 任务中跳过 PLAN，指令跟随率中等的模型在 1/5 中跳过（K2.6: 5/5 跳过；GLM-5.1: 1/5 跳过，T4 constraint decay）。

低指令跟随率模型在全部任务中跳过了 PLAN 阶段。中等指令跟随率模型在最简单的任务中也跳过了。ADR 0008 的安全网假设——"漂移检测已提供安全网"——不成立，因为漂移检测的前提是 PLAN 存在，PLAN 被跳过时安全网也一起消失。

## Decision

引入"双维度可逆性"模型，修正 PLAN 的信任光谱位置：

- **产出可逆性**：操作的产出可以被修改或撤销。PLAN 的产出（计划文档）是可逆的——可以修改步骤、调整约束。
- **执行可逆性**：操作被跳过后，下游的修复成本。PLAN 的执行是不可逆的——跳过 PLAN 导致所有下游失去锚点（漂移检测、约束再注入、TDD 模式声明），修复成本是重做整个任务。

**结论**：PLAN 从"可逆，偏 Hephaestus"（现称偏自主判断）修正为"不可逆，偏指令跟随"。与 DISCOVER 的属性一致——产出可修改，但跳过的后果不可逆。

### 信任光谱修正

| 阶段 | ADR 0008 分类 | 修正后分类 | 理由 |
|---|---|---|---|
| PLAN | 可逆，偏 Hephaestus（现称偏自主判断） | **不可逆，偏指令跟随** | 跳过 PLAN = 下游全部失去锚点，修复成本 = 重做任务 |

其他阶段分类不变。

### 实施方式

将 PLAN 门控升级为 Hard Invariant #5："Never enter EXECUTE without a completed PLAN"。这比 ADR 0008 的"偏 Hephaestus"（现称偏自主判断）定位更强——从"信任模型会自觉执行"变为"禁止跳过"。

### 未修正的项

- **TDD 顺序**：保持"中间"位置。TDD 反转是低指令跟随率模型的局部问题，非协议设计缺陷。交付物质量未受影响。PLAN 门控生效后，PLAN 中的 `[TDD]` 标记可能自然改善 TDD 顺序。
- **QA GATE**：保持"不可逆，偏指令跟随"。QA GATE 跳过的 3 个案例全部伴随 PLAN 跳过，可能是 PLAN 缺失的级联效应。先观察 PLAN 门控效果，再决定是否加因果锁。
- **简单任务防护**：不加对抗性提醒。PLAN 门控已覆盖简单任务场景。
- **跨任务弱点回顾**：暂不加。典型使用模式为单任务会话，跨会话无效。

## Consequences

- ADR 0008 信任光谱表中 PLAN 的行需标记为被本 ADR 推翻
- deepworker.md 新增 Hard Invariant #5 + Immediate Detection 补救行
- 后续新模型测试中，重点观察 PLAN 门控是否有效，以及 QA GATE 跳过是否随之改善
- 如果 PLAN 门控生效后 QA GATE 仍被独立跳过，需新 ADR 评估因果锁方案
