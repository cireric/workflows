# Bias Patterns — Platform & Domain Specific

> 可扩展的区域知识文件。核心规则在 SKILL.md 中，跨领域通用。本文件收录特定平台/领域的偏差识别信号，按需更新。

## 使用方式

按信息源类型定位：社区与社交平台 → 评测与推荐内容 → 学术与标准来源 → 跨平台通用模式。从特化到通用，先查平台特有偏差，再查通用模式。每条偏差模式标注了与 SKILL.md 规则的对应关系。

## 2. 评测与推荐内容

> 以产品/服务评价为核心的信息源。按生产者规模分组——用户生成、专业媒体、行业机构。

### 2.1 用户生成评测

> 电商平台用户评价、社区体验分享。生产者分散、门槛低。

## 4. 跨平台通用模式

> 不绑定特定平台、地域或内容类型。按偏差机制分组。

### 4.1 AI 生成与伪独立验证

> AI 改写使"多源验证"可靠性下降——多个网站发表相似结论看似独立验证，实际同源。

## 扩展指南

添加新平台/领域时，遵循以下格式：

```markdown
### [平台/领域名称]

| 偏差模式 | 识别信号       | 校正方法   | 添加日期 | 规则对应 |
| -------- | -------------- | ---------- | -------- | -------- |
| [模式名] | [可观察的信号] | [如何校正] | YYYY-MM  | [规则N]  |
```

**添加原则**：

1. 只收录实际研究中遇到的偏差模式——无实际失败记录则不添加
2. 识别信号必须是可观察的（文本/结构特征），而非主观判断
3. 校正方法必须是可操作的（具体搜索/对比动作），而非"注意甄别"等空泛建议
4. 每条模式标注添加日期，过时模式定期清理
5. 新增模式若跨平台存在则提升为通用模式（移至第4节或建议提升至 SKILL.md）
6. 校正方法必须本身不受相同偏差影响
7. 新增模式与现有模式可能存在依赖关系（如"联盟营销经济学"是"购买引导"的系统性版本），应标注关联
8. 每条模式必须标注"规则对应"——指向 SKILL.md 抽象规则
9. 校正方法必须是 Researcher 可执行的操作——websearch/webfetch 可访问的平台直接写平台内操作；不可访问的平台用间接操作（搜索同话题在可访问平台的独立讨论、要求用户提供文字转述）

**分类原则**：

- **第1层按信息源类型**：社区 / 评测 / 学术 / 通用
- **第2层按自然分组**：社区按语言；评测按生产者规模（用户/专业媒体/行业机构）；通用按偏差机制
- **新条目归入最具体的分组**；跨多个分组则归入"跨平台通用模式"
