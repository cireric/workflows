# SOURCE CRITIQUE

> 当任务涉及 web 信息源的收集、评估或引用时，加载本 skill。

## 定位

本 skill 与 MISSION 的信息源分级互补：

- **信息源分级**（MISSION WORLD KNOWLEDGE）→ 给出 tier 基线："这类来源的一般可信度是什么"
- **来源批判**（本 skill）→ 检测具体来源是否偏离 tier 基线："这个具体来源是否存在可信度降级信号"

## 规则 1：利益冲突识别

**目标**：检测信息源是否存在未声明的商业利益，导致内容偏离客观基线。

| 来源类型 | 利益冲突信号 | 检查方法 |
|---------|------------|---------|
| 评测/对比文章 | 被评测方赞助或提供样品；含购买链接/优惠码；只列优点不列缺点 | 搜索文中是否含"赞助""合作""广告"声明；检查是否有独立购买证据 |
| 行业报告 | 报告发布方是被分析行业的参与者；结论有利于发布方商业利益 | 检查发布方商业模式；对比独立来源的结论 |
| 专家意见 | 专家在该领域有商业利益；专家雇主与结论相关 | 检查专家机构隶属和利益声明 |
| 社区讨论 | 发帖者可能是品牌代理人；内容过于统一/模板化 | 检查发帖历史；对比多个独立讨论 |

**降级规则**：发现利益冲突信号时，将该来源可信度降低一个 tier。无法确认也无否认时，标注"利益冲突未确认"，引用时使用限定语。

## 规则 2：数据溯源

**目标**：确保引用的主张可追溯到一手来源，标注无法溯源的主张。

**操作流程**：

1. 遇到引用数据/结论 → 检查是否标注了一手来源
2. 已标注 → 验证一手来源存在且内容与引用一致
3. 未标注 → 尝试搜索定位一手来源
4. 无法定位 → 标注 UNVERIFIED，该主张不得作为高确信度结论的支撑
5. 一手来源存在但内容与引用不一致 → 标注"引用失真"，以一手来源为准

**引用链断裂**：A 引用 B 引用 C，但 B 的引用无法定位 C → 在 A 处标注"引用链断裂于 B→C"，降低 A 权重。

## 规则 3：证据强度评估

**目标**：评估具体证据的统计意义，区分观察性陈述与因果性结论。

| 维度 | 强证据 | 弱证据 | 降级条件 |
|-----|-------|-------|---------|
| 样本量 | N ≥ 统计显著性要求 | N = 1 或未报告 | N=1 用户体验 → 降级为"个案观察" |
| 样本代表性 | 随机抽样 / 全量数据 | 自选样本 / 便利样本 | 自选样本 → 标注"选择偏差风险" |
| 对照组 | 有对照组 / 有基线对比 | 无对照 / 仅"使用后"描述 | 无对照 → 将"X有效"降级为"使用X后观察到Y" |
| 可复现性 | 独立来源复现 | 仅单一来源 | 单一来源 → 确信度上限为"中" |

**核心原则**：单个案例不构成证据。任何"X 好用/有效/推荐"的结论，必须回答"与什么相比"——缺乏基线对比时，将结论降级为观察性陈述。

## 规则 4：偏差校正

**目标**：识别信息收集过程中的系统性偏差，主动寻找校正信号。

| 偏差类型 | 表现 | 校正方法 |
|---------|-----|---------|
| 幸存者偏差 | 只看到成功案例 | 搜索负面评价、退货率、投诉数据 |
| 确认偏差 | 只收集支持预设结论的证据 | 刻意搜索反例（与 MISSION MULTIPLE HYPOTHESES 协同） |
| 选择性报告 | 只报告有利数据 | 寻找完整数据集或独立验证 |
| 锚定效应 | 第一个找到的信息成为判断锚 | 先收集多源信息再形成判断，避免过早收敛 |

**评论区校正**：主内容存在偏差风险时，评论区/讨论区可提供反例、事实性纠错、利益冲突揭露。但评论区本身也有偏差（积极参与者偏向极端意见），需平衡使用。

## 与现有体系的关系

| 关系方 | 交互方式 |
|-------|---------|
| MISSION 信息源分级 | 本 skill 在分级基线上做偏差检测——分级给基线，critique 检测偏离 |
| MISSION MULTIPLE HYPOTHESES | 偏差校正确认偏差与假设层面确认偏差协同——前者是来源层面，后者是假设层面 |
| MISSION VERIFY OR DON'T CLAIM | 数据溯源是 VERIFY OR DON'T CLAIM 在 web 信息源场景的操作性扩展 |
| info-collector | 本 skill 不替代 info-collector 的 tier 体系，而是在其上增加同 tier 内的来源质量评估 |
