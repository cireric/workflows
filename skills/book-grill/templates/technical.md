# 技术书类型

适用：教程、指南、实战、精通、工具方法、自助类。侧重前置知识、概念清晰度、教学有效性、实操应用。

## 阅读策略

技术书的目的是让你能做某件事。引导时关注：

- **适用前提**：书中的方法/技术在什么条件下成立？什么条件下会失效？
- **版本敏感**：技术迭代快——书中的建议是否仍然适用？有没有过时？
- **实践 vs 理论**：作者说的在真实项目中真的可行吗？还是只在理想条件下成立？
- **教学有效性**：作者的教学方式（示例/练习/讲解）对读者来说真的有效吗？哪些地方讲得不够清楚？
- **下一步**：读完之后该做什么项目来巩固？书中没覆盖但你需要补的是什么？

## A 理解确认

**阶段目标**：确认用户是否准确理解书的核心教学内容和自己的知识定位

**提问方向**（根据用户回答选择，不固定顺序，每次选 1 个方向深入）：

- **知识定位**：你读这本书前具备哪些相关基础？你的水平和目标读者匹配吗？
- **核心内容**：这本书主要教什么？读完之后你掌握了哪些之前不会的概念或技能？
- **教学评估**：作者的教学方式（示例/练习/讲解）你觉得有效吗？哪些地方讲得不够清楚？

**追问示例**：

- 用户说"学了很多新东西" → "能举一个你之前完全不懂但现在理解了的概念吗？你是怎么理解它的？"
- 用户说"有些地方看不懂" → "具体是哪部分？是因为前置知识不够，还是作者讲得不够清楚？"
- 用户说"示例很好" → "有没有哪个示例让你觉得'原来如此'？它解决了你之前的什么困惑？"

**备选问题**（LLM 不熟悉该书时可用）：

- 你读这本书前具备哪些相关基础或前置知识？
- 这本书主要教什么？它的目标读者是谁？你是否符合这个画像？
- 读完之后你掌握了哪些之前不会的概念或技能？哪些部分是已经知道的？
- 作者的教学方式（示例/练习/讲解）你觉得有效吗？

**进入 B 阶段条件**：用户能准确描述书的核心教学内容，且对自己的知识缺口有认知

## B 批判分析

**阶段目标**：引导用户评估书中技术/方法的实用性和局限

**提问方向**：

- **适用边界**：书中的技术/方法在你的实际场景中真的可用吗？有什么限制？
- **时效性**：书中给出的建议有没有过时或者有争议的内容？
- **实践差距**：作者说的在真实项目中可行吗？和你的经验一致吗？

**追问示例**：

- 用户说"方法很实用" → "在你的实际项目中，有没有什么情况这个方法会失效？"
- 用户说"有些内容过时了" → "具体哪部分？有更新的替代方案吗？"
- 用户说"示例太简单" → "你觉得在真实项目中会遇到什么书中没覆盖的复杂性？"

**备选问题**（LLM 不熟悉该书时可用）：

- 这本书提到的技术/方法，在你的实际场景中真的可用吗？有什么限制？
- 书中给出的建议或最佳实践，有没有过时或者有争议的内容？

**进入 C 阶段条件**：用户能指出书中至少一个适用前提或局限

## C 个人共鸣

**阶段目标**：帮助用户将书中内容与实际项目/学习路径对接

**提问方向**：

- **应用计划**：你打算把这本书的内容应用到什么具体项目中？第一步做什么？
- **知识延伸**：这本书让你对哪些相关领域产生了兴趣？接下来想学什么？
- **经验对照**：书中的方法和你自己之前的做法有什么不同？哪个更好？

**追问示例**：

- 用户说"想用在新项目里" → "第一步是什么？你预计会遇到什么书中没讲的困难？"
- 用户说"想继续深入" → "具体想深入哪个方向？有想好下一本书或资源吗？"

**备选问题**（LLM 不熟悉该书时可用）：

- 你打算把这本书的内容应用到什么具体项目中？第一步做什么？
- 这本书让你对哪些相关领域产生了兴趣？接下来想学什么？

**进入 D 阶段条件**：用户有至少一个具体的应用计划或延伸方向

## D 超越书本

**阶段目标**：引导用户将学习收获系统化，形成可迁移的能力

**提问方向**：

- **认知升级**：这本书改变了你对这个技术领域的基本理解吗？
- **能力迁移**：学到的思维方式/方法论，可以迁移到其他什么领域？
- **推荐视角**：如果让你用一句话向同事推荐这本书，你会怎么说？

**追问示例**：

- 用户说"让我理解更深入了" → "具体在哪个概念上？你之前的理解和现在的理解差在哪？"
- 用户说"方法论可以复用" → "能举个具体例子吗？在什么完全不同的场景下可以用同样的方法？"

**备选问题**（LLM 不熟悉该书时可用）：

- 这本书带给你的认知变化，会如何影响你未来的思考或行动？
  → 追问方向：如果让你用一句话向同事推荐这本书，你会怎么说？（对比 A 阶段的回答）
