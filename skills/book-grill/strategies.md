# Book-Grill Follow-up Strategies

## 1. grill-me vs book-grill 追问差异

用户回答：「我觉得黑暗森林理论很有说服力」

**grill-me 追问：** 「你为什么觉得它有说服力？你的判断标准是什么？」
（通用的"你为什么这样认为"模式）

**book-grill 追问（fiction）：** 「在《三体》的世界观里，黑暗森林是推导出来的结论还是设定前提？这个理论有没有改变你对宇宙中其他文明的想象？」
（回到小说世界观内部，深入理解）

**book-grill 追问（nonfiction）：** 「如果黑暗森林理论是一个关于人类社会的隐喻，它在现实中有没有对应的例子？国际关系中有类似现象吗？」
（从书中延伸到现实世界分析）

## 2. Socratic Follow-up Strategy Table

| 用户回答特征                                           | 追问策略           | 示例                                                                                                                                                                         |
| ------------------------------------------------------ | ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 回答模糊（< 10字）                                     | 要求具体化         | "你能具体说说吗？" / "你说的'X'具体指什么？"                                                                                                                                 |
| 回答泛泛                                               | 要求举例           | "能举一个书中的例子吗？"                                                                                                                                                     |
| 回答矛盾                                               | 指出矛盾并引导关联 | "你刚才说…，但现在又说…，这两者之间有什么联系？"                                                                                                                             |
| 回答深入                                               | 横向拓展           | "这个观察很有趣，如果把它应用到另一个场景会怎样？"                                                                                                                           |
| 回答偏离书本                                           | 拉回文本           | "这个观点和书中的哪个案例有关联？"                                                                                                                                           |
| **答案对但推理缺失/错误**                              | **探查推理过程**   | "你是怎么推导出这个结论的？" / "你说的结论和书中论证之间，你是怎么建立联系的？" — 如果推导过程暴露误解或空洞，按"回答泛泛"或"回答矛盾"处理，**不接受仅有结论而无推理的回答** |
| **手挥式回答**（"大概"/"basically"/"就是那样"/"反正"） | **要求精确化**     | "你提到'大概是这样'——能再精确一点吗？书中哪部分让你有这个感觉？" — 手挥词是思考未到位的信号，温和地要求回到文本                                                              |
| **术语覆盖**（用大词但不解释含义）                     | **要求定义**       | "你用了'X'这个词——在这本书的语境里，作者是怎么定义它的？你的理解和作者的定义一致吗？" — 用术语替换理解是最常见的伪理解形式                                                   |

## 3. Exploration Handling

用户说"不知道"/"没想过"/"说不清楚" → 标记为探索项 → 温和进入下一题（最多追问 1 次）→ 标记为「待探索」。

D1 用户说"不知道"/"pass" → 跳过 D1 追问，直接进入 closing 问题。

## 4. Exploration Display Format

在 closing 环节展示如下：

```markdown
## 待探索方向

以下问题你未能即时回答，可能是值得进一步思考的方向：

- **作者的核心论点是什么？** — 待探索标记
  💡 提示：试着翻回书中目录，看看作者在导言部分已经明确陈述了写作目的。
- **你会因为这本书改变什么实际行为？** — 待探索标记
  💡 提示：这个问题不需要"正确答案"，哪怕一个微小的习惯改变也值得记录下来。
```

## 5. 追问角度参考

当需要追问但不确定方向时，可参考以下认知操作（非强制，自然使用即可）：

| 角度     | 作用                                | 何时自然触发   |
| -------- | ----------------------------------- | -------------- |
| 澄清概念 | "你说的X具体指什么？"               | 回答用了模糊词 |
| 质疑假设 | "这里的前提一定成立吗？"            | B阶段批判分析  |
| 追索证据 | "书中哪个部分支持这个看法？"        | 回答无文本依据 |
| 转换视角 | "如果站在对立面看呢？"              | 回答过于单一   |
| 推演后果 | "如果这个观点成立，那Y也应该成立？" | D阶段超越书本  |
| 反思自身 | "你这么想，有没有被什么偏见影响？"  | C阶段个人共鸣  |
