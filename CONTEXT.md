# Agent System Prompts

Agent 系统提示词的设计与优化——以 agents/researcher.md 和 agents/deepworker.md 为核心对象。

## Agents

**Researcher**:
方法论研究 Agent。理解现实、分析本质、构建知识、设计实验、优化提示词、评估方案。核心：理解而非实现，推理而非编码。
_Avoid_: 执行者、编码者、实现者

**Deepworker**:
深度执行 Agent。给定目标，端到端完成。探索充分后行动，验证后才交付。不半途而废，不跳过验证。核心：有纪律的自主——自主规划执行路径，但遵守补偿指令衰减的协议。
_Avoid_: 编排者、研究者、分析师

## Language

**门控 (Gate)**:
阶段转换时强制执行的填表式审计协议。LLM 的角色是填充答案而非选择攻击方向。
_Avoid_: 检查点 (checkpoint)、审查 (review)

**轻量输出**:
简单查询在标准输出结构中的自然收敛——Hypotheses/Risks 等结构存在但内容精简。不是审计协议的简化，不是可选择的输出模式。
_Avoid_: 轻量模式、简化输出、省略结构

**意图级表述**:
在概述或引用点保留规则的设计意图和方向，不重复操作细节。操作细节指向唯一权威定义点。
_Avoid_: 摘要、缩写

**元认知检查**:
研究者对自身思维过程的审视——锚定检测、复杂化检测、确信度校准。与问题框架级检查（审视外部对象）正交。
_Avoid_: 自省、反思

**回溯依赖**:
模型执行某条规则时，需要跳转到远处查阅完整定义的依赖关系。空间距离越大，注意力衰减越严重，依赖越可能失效。
_Avoid_: 跨节引用、前向引用（过于技术化，不精确）

**停止信号**:
搜索扩展的终止条件。仅适用于中/低重要性结论；高重要性结论的停止条件见 EXHAUSTIVE SEARCH。
_Avoid_: 搜索终止、搜索上限

**设计约束**:
EXECUTOR AWARENESS 的定位——不是一次性前置步骤（做完就过），而是提示词中每条指令的过滤条件（贯穿始终）。每条指令必须通过执行者分析判断必要性，不必要的指令删除或简化。
_Avoid_: 前置分析、执行者画像（仅描述分析动作，未表达过滤功能）

**产物复杂度**:
研究产物的信息量与结构复杂度。与研究过程复杂度正交——过程可以穷举多维度，产物只需包含决策关键变量。现有复杂度控制规则（停止扩展信号、输出边界、复杂化检测、元认知检查）的控制对象是研究过程，产物复杂度是未覆盖的维度。
_Avoid_: 提示词长度、输出大小（过于具体，不表达复杂度的因果来源）

**约束重注入**:
在阶段转换或停滞检测时，强制重新声明关键约束的机制。补偿 K2.6 的长会话指令衰减（WDCD -100% Round 3 衰减）。分三层：硬约束（写死6条）、任务约束（PLAN 声明）、项目特化约束（AGENTS.md 读取）。
_Avoid_: 约束提醒、规则重述（未表达补偿衰减的设计意图）

**TODO 铁律**:
执行阶段的基础纪律——步骤追踪、单步聚焦、完成标记、偏离检测、编辑后即时验证。100% 场景覆盖，始终生效，不可降级。与 TDD 增强层正交——TODO 解决"我在哪、还剩什么"，TDD 解决"代码对不对"。
_Avoid_: 待办事项、任务列表（过于泛化，未表达铁律的强制性）

**有纪律的自主**:
Deepworker 的核心定位——自主规划执行路径，但遵守补偿模型弱点的结构化协议。与 Hephaestus 的"不问就做"不同：Hephaestus 信任模型能力，Deepworker 承认模型局限并主动补偿。
_Avoid_: 半自主、受限自主（暗示自主被削弱，而非自主被增强）
